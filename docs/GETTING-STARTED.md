# Getting Started — Design Token Compiler

**Quick Start Guide for Current Build**

---

## What This Tool Does

The Design Token Compiler solves the **$58,500/month problem** of design-code drift. It's a web app where teams:

1. Define design tokens once (colors, spacing, typography, etc.)
2. Compile them to every platform (CSS, TypeScript, Tailwind, Swift, Kotlin, etc.)
3. Sync between Figma and code with automatic conflict detection

**One source of truth → Every platform stays in sync**

## Understanding the Sample Tokens

The pre-loaded sample follows the **three-tier structure**:

### Tier 1: Primitive Tokens
Raw values, organized by scale:
```
color.primitive.blue.600 = #2563eb
color.primitive.red.500 = #ef4444
spacing.primitive.4 = 16px
```

### Tier 2: Semantic Tokens
What primitives mean in context:
```
color.semantic.primary = {color.primitive.blue.600}
color.semantic.error = {color.primitive.red.600}
spacing.semantic.content-gap = {spacing.primitive.4}
```

### Tier 3: Component Tokens
How semantic tokens are used:
```
color.component.button-primary-bg = {color.semantic.primary}
color.component.card-border = {color.semantic.border}
spacing.component.card-padding = {spacing.primitive.6}
```

**References are resolved automatically:**
- `color.component.button-primary-bg` → resolves to `#2563eb`
- The compiler follows the chain and outputs the final value

---

## Token Reference Syntax

### Simple Reference
```json
{
  "value": "{color.primitive.blue.600}"
}
```
Resolves to: `#2563eb`

### Embedded Reference
```json
{
  "value": "1px solid {color.semantic.border}"
}
```
Resolves to: `1px solid #d1d5db`

### Multiple Embedded References
```json
{
  "value": "{spacing.primitive.2} solid {color.semantic.primary}"
}
```
Resolves to: `8px solid #2563eb`

## Light/Dark Theme Support

Tokens can have mode-specific overrides:

**Light Mode (Default):**
```
color.semantic.background = #ffffff
color.semantic.text-primary = #1a1a1a
```

**Dark Mode (Override):**
```
color.semantic.background = #1a1a1a
color.semantic.text-primary = #ffffff
```

**CSS Output:**
```css
:root, [data-theme="light"] {
  --color-semantic-background: #ffffff;
  --color-semantic-text-primary: #1a1a1a;
}

[data-theme="dark"] {
  --color-semantic-background: #1a1a1a;
  --color-semantic-text-primary: #ffffff;
}
```

## How the Resolver Works

The **token resolution engine** is the heart of the tool. It:

1. **Flattens nested tokens** to dot-notation paths
2. **Applies mode overrides** if a theme is selected
3. **Resolves references** recursively:
   - `{color.semantic.primary}` → looks up → `{color.primitive.blue.600}` → `#2563eb`
4. **Detects errors:**
   - Circular references (`A → B → C → A`)
   - Missing references (`{token.that.doesnt.exist}`)
5. **Handles embedded references:**
   - `"1px solid {color.border}"` → replaces `{color.border}` with resolved value

**Test it:** Check `src/lib/__test-resolver.ts` and run:
```bash
npx tsx src/lib/__test-resolver.ts
```

## Modifying Sample Tokens

To change the sample tokens, edit:
**`src/data/sampleTokens.ts`**

Example: Add a new color
```typescript
'success': {
  id: nanoid(),
  name: 'Success',
  type: 'color',
  value: '{color.primitive.green.600}',
  description: 'Success state color',
}
```

Save the file → The dev server will reload → New token appears in compiler output

## Troubleshooting

### Dev server won't start
```bash
# Kill any process using port 5173/5174
lsof -ti:5173 | xargs kill -9
npm run dev
```

### Can't see sample tokens
- Check browser console for errors
- Clear localStorage: `localStorage.clear()` in console
- Refresh page

### Copy button doesn't work
- Requires HTTPS or localhost (browser security)
- Try download instead

### TypeScript errors
```bash
# Reinstall dependencies
rm -rf node_modules package-lock.json
npm install
```
